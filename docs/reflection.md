# Reflection

## Challenges

- Designing a multi-agent architecture while keeping the initial codebase simple.
- Balancing realtime data ingestion, forecasting, and retrieval-augmented generation in a single prototype.

## Learnings

- FastAPI provides a strong API structure for rapid backend development.
- A modular ML pipeline separates data cleaning, feature engineering, and model persistence.
- Placeholder agent orchestration can be extended to Azure OpenAI or LangChain.

## Next improvements

- Add Azure Data Factory or Databricks ingestion flows.
- Implement a production-grade vector store and Azure OpenAI RAG pipeline.
- Build a Power BI report based on forecasting and anomaly data.
